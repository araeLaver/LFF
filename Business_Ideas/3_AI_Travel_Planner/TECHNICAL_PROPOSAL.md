# Technical Proposal: AI 여행 플래너

## 1. 아키텍처 원칙

*   **데이터 중심 설계:** 추천의 질은 데이터의 질과 양에 비례한다. 양질의 로컬 장소 데이터를 지속적으로 수집, 정제, 갱신하는 데이터 파이프라인을 아키텍처의 핵심으로 삼는다.
*   **빠른 응답 속도:** 사용자가 여행 계획을 요청했을 때, 수십 초 이상 기다리게 해서는 안 된다. 실시간에 가까운 추천을 제공하기 위해, 사전 연산(Pre-computation)과 효율적인 캐싱 전략을 적극적으로 사용한다.
*   **모바일 우선:** 사용자는 여행지에서 스마트폰으로 계획을 확인하고 수정한다. 모든 UI/UX는 모바일 환경에 최적화하여 설계한다.

## 2. 주요 기술 스택

### 2.1. 프론트엔드 (Mobile)
*   **선택:** **React Native 또는 Flutter**
*   **이유:**
    *   하나의 코드 베이스로 iOS와 Android 앱을 동시에 개발할 수 있어, MVP를 빠르게 출시하고 검증하는 데 비용 효율적이다.
    *   지도, 카메라 등 네이티브 기능과의 연동이 용이하다.
    *   활발한 커뮤니티와 풍부한 라이브러리 생태계를 갖추고 있다.

### 2.2. 백엔드 (Backend)
*   **선택:** **Python + FastAPI**
*   **이유:**
    *   웹 크롤링(Scrapy, BeautifulSoup), 데이터 분석(Pandas, Numpy), AI/ML(LangChain, Scikit-learn) 등 데이터 중심 애플리케이션 개발에 필요한 라이브러리 생태계가 매우 강력하다.
    *   추천 시스템, 자연어 처리 모델을 서빙하기에 적합하다.
    *   FastAPI의 비동기 처리 능력은 다수의 외부 API(지도, 소셜미디어 등)를 호출하고 데이터를 조합해야 하는 서비스에 높은 성능을 보장한다.

### 2.3. 데이터베이스
*   **선택:** **PostgreSQL + PostGIS, MongoDB**
*   **이유:**
    *   **PostgreSQL (+PostGIS 확장):** 사용자의 프로필, 여행 계획 등 정형 데이터를 저장하고, PostGIS를 통해 장소의 위도/경도 등 위치 기반 데이터를 효율적으로 쿼리(예: "반경 500m 내의 카페 찾기")하는 데 사용한다.
    *   **MongoDB:** 블로그, SNS 등에서 수집한 비정형 텍스트/이미지 데이터를 유연하게 저장하고 검색하는 데 사용한다.

### 2.4. AI / 추천 시스템
*   **접근 방식:** **하이브리드 추천 시스템 + LLM**
    1.  **데이터 수집:** Scrapy, BeautifulSoup 등을 이용해 양질의 여행 블로그, 인스타그램, 로컬 커뮤니티 등의 데이터를 주기적으로 크롤링하여 MongoDB에 저장한다.
    2.  **데이터 정제/임베딩:** 수집된 데이터에서 장소명, 주소, 카테고리, 리뷰 텍스트 등을 추출한다. 텍스트 데이터는 **임베딩 모델(Sentence-BERT 등)**을 통해 벡터로 변환하여 벡터 DB(e.g., Pinecone, Chroma)에 저장한다. 이는 '분위기', '느낌' 등 시맨틱 검색을 가능하게 한다.
    3.  **1차 필터링:** 사용자의 명시적 요구(예: "채식 식당")에 따라 PostgreSQL에서 후보 장소들을 필터링한다.
    4.  **2차 랭킹 (추천 모델):** 필터링된 후보군을 대상으로, 사용자의 암시적 취향(과거 피드백 등)과 장소의 벡터 유사도를 계산하여 최종 추천 순위를 매긴다.
    5.  **여행 계획 생성 (LLM):** 랭킹된 장소 목록을 프롬프트와 함께 LLM(GPT, Gemini)에 전달하여, 자연스러운 문장과 동선, 팁이 포함된 최종적인 '여행 계획 텍스트'를 생성한다.

## 3. 핵심 연동 API
*   **지도 API:** Google Maps API, Naver/Kakao Maps API (위치 정보, 길찾기, 장소 상세 정보)
*   **소셜 미디어 API:** Instagram Basic Display API 등 (보조적인 데이터 수집)
*   **예약 API:** Skyscanner, Agoda, Klook 등 (향후 제휴 수익 모델을 위한 연동)
